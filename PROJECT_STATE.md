## 🗄️ Архитектура моделей

### Category (Категория)
- name (CharField) - название
- slug (SlugField) - URL
- parent (ForeignKey self, null=True) - родительская категория (для вложенности)
- description (TextField) - описание для главной страницы
- order (IntegerField) - порядок сортировки
- is_active (BooleanField) - видимость
- created_at, updated_at

### Product (Товар)
- title (CharField) - название
- article (CharField, blank=True) - артикул
- slug (SlugField) - URL
- category (ForeignKey Category) - категория
- subcategory (ForeignKey Category, null=True) - подкатегория
- description (TextField) - подробное описание
- details (JSONField) - характеристики {"Производитель": "Antari", ...}
- preview_image (ImageField) - главное фото
- availability (CharField) - "В наличии" / "Под заказ"
- is_active (BooleanField) - видимость на сайте
- views_count (IntegerField) - счетчик просмотров
- created_at, updated_at

### ProductImage (Дополнительные изображения)
- product (ForeignKey Product)
- image (ImageField)
- order (IntegerField) - порядок отображения
- created_at

### ContactRequest (Запрос КП)
- name (CharField) - имя
- organization (CharField, blank=True) - организация
- phone (CharField) - телефон
- email (EmailField, blank=True)
- request_type (CharField) - тип запроса: "КП", "Консультация", "Поддержка"
- product (ForeignKey Product, null=True) - товар (если из карточки)
- message (TextField) - сообщение
- created_at
- is_processed (BooleanField) - обработано

## 📝 Формат импорта данных

### Поля CSV:
- title - название товара
- article - артикул (может быть пустым)
- category - категория
- price - цена (не используем, у нас КП)
- availability - наличие: "Наличие:В наличии" / "Наличие:Под заказ"
- description - описание
- details - характеристики в формате "Производитель: Antari | Наименование: <h3>AF-3</h3> | ..."
- images - URL изображений через пробелы/табы
- url - URL товара на risbar.kz (не используем)

### Логика импорта:
1. Парсинг файла (определение формата по расширению)
2. Создание/обновление категорий
3. Парсинг поля details → JSON
4. Загрузка изображений по URL
5. Установка preview_image (первое изображение)
6. Создание ProductImage для остальных
7. Очистка HTML-тегов из характеристик
